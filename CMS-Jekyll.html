Content Management System and Jekyll new terms

Content Management Application (CMA): Part of the two main components of a CMS. CMA is the frontend interface through which the user interacts. This components handles the content creation, editing, and management tasks. For instance, the user can create a new blog post, edit an existing one, or delete content as needed.

Content Delivery Application (CDA): Part of the two main components of CMS. CDA is the backend interface through which the content is delivered to the end user. It handles the storage, retrieval, and delivery of content, ensuring that the content created and managed via the CMA is properly presented to the audience. Is the part of the CMA that handles the compilation of the website contents and its updates.

Jekyll: Is a static site generator. This means it takes raw content, applies templated, and generates static HTML files that can be served to users. It is called static because the generated HTML files do not change dynamically based on user interactions, they remain the same until the site is rebuilt.
