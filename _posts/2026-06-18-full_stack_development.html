Full Stack Development

Full Stack Development Steps

Introduction

Full stack development is an art in the sense that there is no a single formula or process in which it must be done, people can start and end at any part of the process. I will try to focus in the general steps that are somewhat neccessary for good website development.

Defining Requirements

What is often the very first step of full stack development is defining our functional and non-functional requirements. In this step, we define what are the items our application must do (functional requirements) and some constraints (non-functional requirements). This is often the first document that defines our program.

Normally we can define the functional requirements in a table format, listing each requirements along with its level, the level of the requirement indicates when the requirement must be complete in a given time frame. Here is the table:

Requirement Level Description
Login Functionality 1 The application shall allow users to log in using their credentials.
Registration Functionality 2 The application shall allow users to register with their email and password.

Determining Technologies

One of the earlier steps of full stack development is determining the technologies (libraries, api, etc) that will be needed to achieve the functional requirements of our application. Defining our technologies gives us better scope of our programs. This part of the process helps us determine what are the things that need to be built and what can be reused from other sources.

Testing Strategy

Testing is one of the most important part of full stack software development. Although optional, it is an neceessity for good software development. Once our software development strategy is determined, we can determine our testing strategy. Testing trhoughout our software stack is incredibly beneficial.

For example, if a system has a database connected to it with the ORM models, the ORM models might be one of the first places where an error can occur. Having unit tests for each module of the program could save a lot of debugging time later. Testing also serves as documentation for the expected behavior of function or on this case of the attributes of the ORM models, ensuring that any changes to the code do not break existing functionality. It also could prevent us from making wrong assumptions about the behavior of the code.

Building the database

Although optional, many full stack applications require a database. Building the database is a crucial step in full stack development. It involves designing the database schema, creating tables, defining relationships, and ensuring data integrity.

ORM (Object-Relational Mapping)

You can, at this step, use an existing ORM library such as SQLAlchemy for python, or build your own ORM. Regardless of which you choose, at this step we define the database tables in a way that allows us to interact with the database using objects and manipulate the data in the database using objects instead of writing raw SQL queries.

Service Logic

The service logic layer is responsible for implementing the core functionality of the application. It handles the business rules, processes user inputs, interacts with the database through the ORM, and ensures that the application behaves as expected. For instance: handling user authentication, handling updates to user profiles or processing other users requests based on business rules.
